;===== Reglas de simplificación (rs) =====
;
;Estas palabras deben estar entre [*rs*] y [/rs]
;Se usarán para asociar palabras con claves definidas.
;La idea es poder agrupar palabras diferentes con un mismo significado que
;será el que se incluya en las claves.
;
;Cuando se encuentre [*simp*] (que debe estar antes del bloque de cierre [/rs]
;se añadirán a la lista de palabras a usar para sustituir las palabras en la
;cadena inicial introducida por el usuario.
;Lo que se espera encontrar aquí es el equivalente a lo que Eliza tendría que
;decir, por ejemplo, si el usuario escribe 'tengo', Eliza debe usar 'tienes'
;Por tanto estas palabras sólo se usarán, junto con los verbos, cuando haya
;que usar el texto introducido por el usuario como parte de la respuesta de
;Eliza, es decir cuando en la respuesta de Eliza esté *RESTO*
;
;---Este texto que viene a continuación es lo que puse originalmente y sigue
;   siendo válido para las palabras que estén entre [*simp*] y [/rs]
;
;Estas palabras se usarán para sustituirlas en la frase entrada por el usuario
;y se usarán en la respuesta o bien servirán para buscar en las claves.
;También servirán para sustituir palabras escritas por el usuario para buscar
;entre las claves, por ejemplo, en el caso de los idiomas.
;
;Se debe tener en cuenta que en las palabras claves
;se buscará lo que esté en la segunda parte (después de la coma)
;y las palabras que se incluirán en la respuesta también serán las que estén
;después de la coma.
;
;Se usará: "@palabra,otra" para los casos que existan las dos combinaciones
;palabra,otra y otra,palabra
;
;En las palabras incluidas en [*rs*] sólo se deberían convertir 'frases'
;de dos palabras como máximo, ya que al 'simplificar' se miran las palabras
;de dos en dos. Por tanto: 'todo el mundo,todos' no tendría validez.
;

;===== Claves y sus correspondientes respuestas =====
;
;Las claves estarán entre cochetes y después el Nivel
;Las claves con un nivel más alto se usarán antes que las de menor nivel.
;
;Cada clave principal puede tener subclaves, estas se deberán especificar
;entre [*extras*] y [/extras]
;
;Las palabras en las claves serán las que el usuario ha escrito o el resultado
;de usar las reglas de simplificación incluidas entre: [*rs*] y [*simp*]
;
;---Antes las claves no era lo que el usuario escribía, sino el resultado
;   de haberlas convertido, pero ahora no es así.
;
;Si en la respuesta se usa: *equal:=xxx
;quiere decir que se usará lo que haya en 'LA CLAVE PRINCIPAL' "xxx"
;esto se puede usar en el caso de que se deban dar las mismas respuestas
;en claves diferentes, o bien para que una vez agotadas las respuestas
;de una clave determinada se quiera continuar con las respuestas de otra clave
;
;   "xxx" debe ser una clave principal, no una subclave, aunque se puede poner
;   en cualquier sitio, incluso en las respuestas de las subclaves
;
;El *equal:=xxx se puede usar para llamar a una subclave, aunque debe ser una palabra
;normal, es decir que no contenga {* ...}
;
;Se puede usar [clave {* ...}] para los casos en los que la palabra clave
;vaya seguida de cualquiera de las palabras indicadas entre {* y }
;   Las palabras a tener en cuenta deberán ir separadas por ;
;   de esta forma se permiten palabras clave formadas por varias palabras
;Las respuestas que se usarán para estas palabras serán las que vengan a 
;continuación, ya que internamente se hará es un *equal:= a la primera clave.
;
;Es decir si el analizador se encuentra con: [tu {* hermano; hermana}]
;las respuestas que haya a continuación serán las que se pongan en:
;[tu hermano] y [tu hermana] tendrá como respuesta: *equal:=tu hermano
;
;Para ver ejemplos, busca {*

;Se puede usar {*iif( para usar la respuesta del usuario, según la respuesta
;se usará una parte u otra de lo indicado después de *iif(...;

;El iif funciona igual que en VB, (o casi)
;{*iif(*CONDICIÓN*; ES-TRUE)(ES-FALSE)}
;   si se cumple la condición se usa lo que haya a continuación del punto y coma hasta el siguiente ), 
;   en otro caso se usará lo que esté después de ')('
;   Prefiero usar los paréntesis para poder incluir en la respuesta el punto y coma.
;
;Por ahora la *CONDICIÓN* sólo será:
;   *afirmativo* o *true*   Para los casos en que responda afirmativamente
;   *negativo* o *false*    Para los casos en que responda negativamente
;
;Hay que tener en cuenta de que si se usa: {*iif(*false*; UsarA)(UsarB)}
;Se usará UsarA en caso de que responda "negativamente" y
;se usará UsarB en caso de que responda "afirmativamente"

;He ampliado el uso de {* ; para especificar distintas claves, de forma que si
